# 📘 Cross‑Language Interop Guide  

**Status**: ✅   
**Version**: 0.2.0   
**Last Updated**: 2026.04.07

---

This document provides practical examples for consuming the **C‑Abi‑Bridge‑Aot** NativeAOT library from various programming languages.  

All examples use the exported C ABI functions defined in `API.md`.

---

## 1. Overview

The library exposes a stable C ABI and can be consumed from any language that supports:

- `extern "C"`
- `__cdecl` calling convention
- Flat symbol names (no mangling)

Supported languages include:

- C / C++
- C#
- VB.NET
- Rust
- Go
- Zig
- Python (ctypes / cffi)
- Java (JNI)
- Many others

## 2. C++ Interop

### 2.1 Linking the Import Library

Add to your `.vcxproj`:

```xml
<AdditionalLibraryDirectories>$(SolutionDir)Build\Artifacts</AdditionalLibraryDirectories>
<AdditionalDependencies>C-Abi-Bridge.Aot.lib</AdditionalDependencies>
```

### 2.2 Example Usage

```cpp
#include "bridge-aot.h"

int main()
{
   uint8_t buffer[32];
   cerror err = fill_crypto_bytes_aot(buffer, 32);
   assert_error(err);
}
```

## 5. Rust Interop

### 5.1 FFI Declaration

```
#[link(name = "C-Abi-Bridge.Aot")]
extern "C" {
    fn fill_crypto_bytes_aot(buffer: *mut u8, size: i32) -> CError;
}
```

### 5.2 Example Usage

```
#[repr(i32)]
#[derive(Debug, Copy, Clone, PartialEq, Eq)]
pub enum CError {
    Ok = 0,
    NullPointer = 1,
    InvalidSize = 2,
    // ... weitere Fehlercodes
}

extern "C" {
    fn fill_crypto_bytes_aot(buffer: *mut u8, size: i32) -> CError;
}

fn assert_error(err: CError) {
    if err != CError::Ok {
        panic!("fill_crypto_bytes_aot failed: {:?}", err);
    }
}

fn main() {
    let mut buffer = [0u8; 32];

    unsafe {
        let err = fill_crypto_bytes_aot(buffer.as_mut_ptr(), buffer.len() as i32);
        assert_error(err);
    }

    println!("Generated bytes: {:?}", buffer);
}
```

## 6. Go Interop (cgo)

### 6.1 cgo Header

```
/*
#cgo LDFLAGS: -LC:/path/to/Artifacts -lC-Abi-Bridge.Aot
#include <stdint.h>

int fill_crypto_bytes_aot(uint8_t* buffer, int32_t size);
err := C.fill_crypto_bytes_aot((*C.uint8_t)(&buf[0]), C.int(len(buf)),)
assertError(err)
*/
import "C"
```

### 6.2 Example Usage

```
/*
#cgo LDFLAGS: -lC-Abi-Bridge.Aot
#include "C-Abi-Bridge.Aot.h"
*/
import "C"
import "fmt"

func assertError(err C.CError) {
    if err != C.CError_Ok {
        panic(fmt.Sprintf("fill_crypto_bytes_aot failed: %d", err))
    }
}

func main() {
    buf := make([]byte, 32)

    err := C.fill_crypto_bytes_aot(
        (*C.uint8_t)(&buf[0]),
        C.int(len(buf)),
    )
    assertError(err)

    fmt.Println("Generated bytes:", buf)
}
```

## 7. Zig Interop

Zig can import the `.def` file directly:
```zig
const std = @import("std");
const cabi = @cImport({
    @cInclude("C-Abi-Bridge.Aot.h");
});
```

### 7.1 Example Usage

```zig
const std = @import("std");
const cabi = @cImport({
    @cInclude("C-Abi-Bridge.Aot.h");
});

fn assertError(err: cabi.CError) void {
    if (err != cabi.CError_Ok) {
        @panic("fill_crypto_bytes_aot failed");
    }
}

pub fn main() void {
    var buf: [32]u8 = undefined;

    const err = cabi.fill_crypto_bytes_aot(&buf, buf.len);
    assertError(err);

    std.debug.print("Generated bytes: {any}\n", .{buf});
}
```

## 8. Python Interop (ctypes)

### 8.1 Python (ctypes)

```
from ctypes import *

# Load the native library
dll = CDLL("C-Abi-Bridge.Aot.dll")

# Define CError enum (must match the C header)
class CError(c_int):
    CError_Ok = 0
    CError_NullPointer = 1
    CError_InvalidSize = 2
    # ... additional error codes

# Define function signature
dll.fill_crypto_bytes_aot.restype = CError
dll.fill_crypto_bytes_aot.argtypes = [POINTER(c_uint8), c_int]

def assert_error(err):
    if err != CError.CError_Ok:
        raise RuntimeError(f"fill_crypto_bytes_aot failed: {err}")

# Allocate buffer
buf = (c_uint8 * 32)()

# Call function
err = dll.fill_crypto_bytes_aot(buf, 32)
assert_error(err)

print("Generated bytes:", list(buf))
```

### 8.2 Python (cffi)

```
from cffi import FFI
ffi = FFI()

# Define the C interface
ffi.cdef("""
    typedef enum {
        CError_Ok = 0,
        CError_NullPointer = 1,
        CError_InvalidSize = 2
        // ... additional error codes
    } CError;

    CError fill_crypto_bytes_aot(uint8_t* buffer, int size);
""")

# Load the library
lib = ffi.dlopen("C-Abi-Bridge.Aot.dll")

def assert_error(err):
    if err != lib.CError_Ok:
        raise RuntimeError(f"fill_crypto_bytes_aot failed: {err}")

# Allocate buffer
buf = ffi.new("uint8_t[32]")

# Call function
err = lib.fill_crypto_bytes_aot(buf, 32)
assert_error(err)

print("Generated bytes:", list(buf))
```

## 9. Java Interop (JNI)

```
public class Crypto {
    static {
        System.loadLibrary("C-Abi-Bridge.Aot");
    }

    public static native int fill_crypto_bytes_aot(byte[] buffer, int size);

    private static void assertError(int err) {
        if (err != 0) {
            throw new RuntimeException("fill_crypto_bytes_aot failed: " + err);
        }
    }

    public static void main(String[] args) {
        byte[] buffer = new byte[32];

        int err = fill_crypto_bytes_aot(buffer, buffer.length);
        assertError(err);

        System.out.println("Generated bytes:");
        for (byte b : buffer) {
            System.out.printf("%02X ", b);
        }
    }
}

JNIEXPORT jint JNICALL Java_Crypto_fill_1crypto_1bytes_1aot
  (JNIEnv* env, jclass cls, jbyteArray buffer, jint size) {

    jbyte* ptr = (*env)->GetByteArrayElements(env, buffer, NULL);
    CError err = fill_crypto_bytes_aot((uint8_t*)ptr, size);
    (*env)->ReleaseByteArrayElements(env, buffer, ptr, 0);

    return err;
}
```

## 10. Notes

- All examples assume the DLL is in the process working directory.
- C++ requires the .lib file from Build/Artifacts.
- Other languages load the DLL directly.
- All functions return 0 on success and negative values on error.
